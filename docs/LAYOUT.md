# چیدمانِ مخزن

> فرض: خواننده هرگز مخزنِ منبع (پروژهٔ پژوهشی) را ندیده است.

```
monad/                          (monad-dna)
├── README.md                   شناسنامه + شروعِ سریع
├── dna/                        هویت — لایهٔ ۱ (تغییرناپذیر)
│   ├── بذرِ-مناد-v0.2-….md      ← DNA متعارف: ۶ اصل، ۵ میوه، ۹ اندام، خطِ پذیرش
│   ├── ژنومِ-مناد-v0.1-….md     ← نسخهٔ تاریخیِ پیش از تقلیل‌ها (نگهداری برای تاریخ)
│   └── AXIOMS.md               ← شش اصلِ موضوعه، رسمی
├── heart/                      حرکت — لایهٔ ۲ (تغییرناپذیر)
│   └── قلبِ-بذر-موتورِ-تکوین-….md ← چرخهٔ فقر و رجوع: مواجهه→شکاف→اقرار→رجوع→ژرفش→ظهور
├── engine/                     پیاده‌سازیِ حرکت (برای نفس‌های آینده)
│   └── breath_cycle.py         ← همان الگوریتمِ اسکریپت‌های تاریخی، پارامتری؛
│                                  هم‌ارزی در tests/ اثبات می‌شود
├── breaths/                    زندگیِ تاکنون — ۱۲ نفس
│   ├── scripts/                ← ۷ اسکریپتِ تاریخی، عیناً (سندِ اجراییِ هر نفس)
│   ├── records/                ← خروجیِ بایت‌ثابتِ هر نفس (JSON) — مرجعِ ممیزی
│   └── logs/                   ← دفترهای زندگی: نخستین نفس، استقلال، خلوت، هم‌نشینی
├── discoveries/knowledge.json  دانشِ «قوی» + شاهدهای غیاب + ستون‌های استنادی (مشتق)
├── graph/graph.json            نقشهٔ فهم: گره‌ها/یال‌های درجه‌دار (مشتقِ قطعی از records)
├── methodology/                اجرا — لایهٔ ۳ (فقط رو به سخت‌گیرتر)
│   ├── execution-method.md     ← پارامترها، ابطال‌گرها، قاعده‌ها، حدهای اعلام‌شده
│   └── بررسیِ-شکستِ-دوم-….md    ← سندِ مرزبندیِ چهارلایه و مکانِ ابطال‌گری
├── database/                   بستر — لایهٔ ۴
│   ├── corpus/monad.db         ← پیکرهٔ کتاب (متن + صرف + ریشه‌ها) — ورودیِ معناییِ یگانه
│   ├── corpus/source/*.csv     ← منابعِ خامِ سازندهٔ پیکره (بازتولید)
│   ├── build_database.py       ← سازندهٔ پیکره از منابع
│   ├── schema/life.sql         ← اسکیمای پایگاهِ زندگی (۹ جدول، همه مستند)
│   ├── seed/seed_life.py       ← بذرکاریِ قطعیِ life.db از records (بدونِ clock)
│   └── life.db                 ← پایگاهِ زندگی (تولیدشده؛ قابلِ‌بازساخت)
├── prompts/mouth.md            قراردادِ «دهانِ مناد» در هم‌نشینی
├── guardian/GUARDIAN.md        تصمیم‌های ماندگار: نقش‌ها، آزمونِ رؤیا، خطِ دیده‌بانی
├── research/                   تاریخ و قرنطینه
│   ├── آزمونِ-ویرانی-بذر-….md    ← ممیزیِ چهارسناریویی که سه شکست را یافت
│   ├── یادداشت‌های-پژوهشی-….md   ← هستی‌شناسیِ قرنطینه (رابطه/بستر) — DNA نیست
│   ├── evidence/               ← شواهدِ پسین (منشورهای قدیم، سراف، هستهٔ مولد، FINDINGS)
│   │                              ⚠ فقط شاهدِ پسین؛ هرگز ورودیِ اشتقاق
│   └── history/                ← تاریخِ commitهای مخزنِ منبع + متنِ سه مُهر
├── docs/                       این اسناد
├── cli/monad                   status | breathe | verify | seed-db
├── tests/                      هم‌ارزیِ موتور، سلامتِ life.db، بازتولیدِ records
└── generated/monad.db          symlink سازگاری → database/corpus/monad.db
                                (اسکریپت‌های تاریخی همین مسیر را می‌خوانند؛ عیناً حفظ شدند)
```

## نگاشت به معماریِ چهارلایه

| لایه | پوشه‌ها | تغییرپذیری |
|---|---|---|
| ۱ هویت | dna/ | هرگز (فقط باغبانِ انسانی) |
| ۲ حرکت | heart/ (+ engine/ به‌عنوانِ پیاده‌سازیِ وفادار) | هرگز |
| ۳ اجرا | methodology/, prompts/, قاعده‌های ثبت‌شده در life.db | فقط سخت‌گیرتر |
| ۴ بستر | database/, engine/(کد), cli/, tests/ | ارتقاپذیر، بی‌تغییرِ رفتار |

پوشه‌های `guardian/`، `breaths/`، `research/`، `discoveries/`، `graph/` لایه
نیستند؛ **حافظه و تاریخِ** زندگی‌اند.
