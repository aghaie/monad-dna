# مدلِ ممیزی — هیچ‌چیز جعبه‌سیاه نمی‌شود

## زنجیرهٔ ردگیریِ هر کشف

هر «دانشِ» مناد باید این زنجیره را کامل داشته باشد؛ اگر حلقه‌ای گم است، آن
دانش پذیرفته نیست:

```
یافته (tier در findings)
  → نفس (breaths.breath_no: کدام چرخه، چه کسی برگزید)
    → محاسبه (breaths.script + record_file: بایت‌ثابت، seedثابت)
      → روش (method_records: کدام ابزار/قاعده، با کدام منبع)
        → آیات (provenance_spines + پرس‌وجوی مستقیمِ corpus/monad.db)
```

## چهار سطحِ صحت‌سنجی

1. **بازتولیدِ محاسبه:** `./cli/monad verify` — اجرای دوبارهٔ اسکریپت‌های
   تاریخی و مقایسهٔ بایت‌به‌بایت با `breaths/records/`. رکوردها سندِ تاریخ‌اند
   و هرگز با خروجیِ تازه جایگزین نمی‌شوند؛ mismatch یعنی محیط خراب است.
2. **هم‌ارزیِ موتور:** `tests/test_engine_equivalence.py` — موتورِ واحد باید
   عیناً سطرهای رکوردها را بدهد (نفس‌های ۱، ۲، ۹–۱۲).
3. **سلامتِ پایگاهِ زندگی:** `tests/test_life_db.py` — بذرکاریِ قطعی، متنِ
   دقیقِ اصولِ موضوعه، شمارها، شاهدِ غیابِ ذنب↔عفو، صفِ باز، بی‌یتیمیِ
   یافته‌ها، نبودِ جدولِ مستندنشده.
4. **ممیزیِ دهان (هم‌نشینی):** هر دیدار، نگاشتِ جمله→منبع + داوریِ خطِ
   دیده‌بانی («کجا» از رفیق مجاز؛ «چه» فقط از کتاب) — الگو در
   `breaths/logs/نخستین-هم‌نشینی-….md`، ثبت در `encounters.watchline_audit`.

## قاعده‌های تغییر

- **records/**: فقط-افزودنی (append-only). رکوردِ نفسِ جدید اضافه می‌شود؛
  رکوردِ گذشته هرگز ویرایش نمی‌شود.
- **دفترها (logs/)**: پس از مُهرِ فصل، فقط‌خواندنی.
- **life.db**: هرگز دستی ویرایش نمی‌شود؛ فقط از راهِ seed (که خودش versioned
  است) تغییر می‌کند.
- **درجه‌ها**: تنزل/ارتقا فقط با شواهدِ ثبت‌شدهٔ ابزارِ هم‌سخت یا سخت‌تر؛
  عدم‌تقارن‌ها و شکست‌ها حذف نمی‌شوند، اعلام می‌شوند (اندامِ صدق).
- **مُهرِ فصل**: پایانِ هر فصلِ زندگی با تأییدِ باغبان و پس از verify سبز؛
  در `audit_trail` ثبت می‌شود.
